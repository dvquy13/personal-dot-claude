# Hacker News Platform Reference

## Format Constraints

- **Title**: 80 characters max. No marketing language, clickbait, excessive punctuation, or ALL CAPS.
- **Submission type**: Link to the primary source (your blog post or GitHub repo). If text-only, no upvote button appears — link posts generally get more traction.
- **First comment**: Post this as your own first reply immediately after submitting. It's where you explain the project in your own words.

## Title Rules

For a personal project: use "Show HN: [what it does]" format.
- Good: `Show HN: qrec – local session recall for Claude Code` (47 chars)
- Good: `Show HN: A tool to retrieve past Claude Code sessions locally` (61 chars)
- Bad: `Show HN: I Built an Amazing Tool That Remembers Everything!!!` (hype)
- Bad: `Show HN: qrec` (too vague — no context)

The title should tell a smart person exactly what it does, with no embellishment.

## First Comment Structure

Post this immediately after submission as your own reply:

```
[What it does] — one sentence, plain English.

[Why it exists] — the actual problem it solves, stated plainly. Be honest about the limitation it addresses.

[Try it] — GitHub link or install command. Lower the barrier to trying it.

[Optional: one honest caveat] — "It's early and [X] isn't solved yet" builds trust.

[Open question for the community] — "I'm curious if [specific question]" invites real discussion.
```

Keep it to 3–5 sentences. HN readers are fast and will skim.

## Timing

Post **Tuesday–Thursday, 8–11 AM US Eastern Time** for maximum visibility. The front page is most active then.

## Engagement

- Monitor the thread and respond within 60 minutes of posting. Early activity signals quality to the algorithm.
- Respond to every comment, including critical ones. Dismissive replies kill credibility.
- If someone points out a flaw, acknowledge it honestly. HN respects intellectual honesty.
- Do NOT ask friends, Discord members, or Twitter followers to upvote. Vote manipulation gets votes nullified and your account flagged.

## Disclosure

State clearly in the first comment that you built this. Transparency is mandatory. Trying to look like a neutral third party sharing a cool project is considered deceptive.

## Anti-Patterns

- Marketing language in the title or first comment
- Asking anyone anywhere to upvote
- Reposting the same project under a different angle
- Submitting without a working demo or installable version
- Defensive or dismissive responses to criticism
- Text-only posts (no URL) — they start at a disadvantage

## Output Checklist

- [ ] Title is ≤80 chars
- [ ] Title uses "Show HN:" format
- [ ] No hype language in title
- [ ] First comment is 3–5 sentences
- [ ] First comment includes what it does, why, and a try-it link
- [ ] Timing reminder included
- [ ] Disclosure note included
