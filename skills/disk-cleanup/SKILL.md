# Disk Cleanup

Help the user reclaim disk space on macOS. The workflow is: **scan → propose → confirm → execute → report**.

The key discipline: complete the full scan before proposing anything. Discovering a large item (like Docker volumes) after you've already started deleting is avoidable — run all discovery in one pass first.

---

## Step 1: Full scan (one pass, all parallel)

Run all of these in a single message. Capture disk state and all candidate targets before touching anything.

```bash
# Disk state (APFS — use Data volume, not /)
df -H /System/Volumes/Data | tail -1

# Chrome running? (affects whether cache is deletable)
pgrep -x "Google Chrome" && echo "CHROME_RUNNING" || echo "CHROME_NOT_RUNNING"

# Top-level home dirs
du -sh ~/* ~/.[^.]* 2>/dev/null | sort -rh | head -25

# Key Library subdirs
du -sh \
  ~/Library/Caches \
  ~/Library/Containers \
  ~/Library/Developer/CoreSimulator \
  ~/.npm ~/.pnpm ~/.local \
  2>/dev/null | sort -rh

# Cache breakdown (top 20)
du -sh ~/Library/Caches/* 2>/dev/null | sort -rh | head -20

# Containers breakdown (Docker lives here)
du -sh ~/Library/Containers/* 2>/dev/null | sort -rh | head -10

# Docker volumes (graceful skip if daemon not running)
docker system df 2>/dev/null || echo "DOCKER_NOT_RUNNING"

# Project artifacts — find + size in one pass
# Replace PROJECT_ROOT with actual dir (default: ~/frostmourne)
find PROJECT_ROOT -maxdepth 4 \
  \( -name "node_modules" -o -name ".venv" -o -name "venv" \
     -o -name ".next" -o -name "dist" -o -name "build" \) \
  -prune -exec du -sh {} \; 2>/dev/null | sort -rh

# Git staleness for all projects under PROJECT_ROOT
for dir in PROJECT_ROOT/*/; do
  last=$(git -C "$dir" log -1 --format="%ai" 2>/dev/null || echo "no-git")
  echo "$(basename $dir): $last"
done
```

**PROJECT_ROOT**: default `~/frostmourne`. If the user works elsewhere, ask or infer from context before running.

After this pass, drill into surprising large items with targeted `du -sh` only if needed.

---

## Step 2: Categorize findings

Group everything into these buckets. The goal is to give the user a clear, sized proposal — not to overwhelm with raw numbers.

### Always safe to delete (auto-regenerated by apps)
- `~/Library/Caches/*.ShipIt` dirs (Ollama, VSCode, Discord, Notion, etc.)
- `~/Library/Caches/SiriTTS`
- `~/Library/Caches/ms-playwright`, `ms-playwright-go`
- `~/Library/Caches/claude-cli-nodejs`
- `~/.npm` → `npm cache clean --force`
- `~/.pnpm` → `pnpm store prune`
- Homebrew cache → `brew cleanup`
- Chrome cache (`~/Library/Caches/Google`) — **only if Chrome is not running**

### Stale project artifacts (90-day rule)
For `node_modules`, `.venv`, `venv`, `.next`: if last git commit > 90 days ago → safe to delete (regenerable with `npm install` / `pip install`). No git → treat as stale.

For `dist` and `build`: flag these separately — some projects have no rebuild script or treat these as source. Propose deletion but note the risk and let the user decide per-project.

### iOS simulators
- Old/unavailable via `xcrun simctl delete unavailable` — always safe.

### Docker
- **Volumes** (`docker volume prune -f`): safe if all shown as inactive (ACTIVE = 0). Check `docker system df` output.
- **Images** (`docker image prune -a -f`): much more aggressive — removes all untagged and unused images. Propose separately with a note that re-pulling takes time. Only do this if user explicitly confirms.
- If daemon not running: show estimated size from `~/Library/Containers/com.docker.docker/Data/vms` and note "Start Docker Desktop first to clean Docker volumes (~XG)".

### Needs manual review (never auto-delete)
- `~/Downloads`, `~/Movies`, `~/miniconda3`, `~/.local` — flag sizes, let user decide.

## Step 3: Propose

Present a table with sizes per item and totals per category. Show grand total estimated savings.

Example format:
```
Category                          Item                    Size
─────────────────────────────────────────────────────────────
Safe caches                       *.ShipIt dirs           2.3G
                                  SiriTTS                 479M
                                  Playwright caches       1.9G
                                  npm/pnpm/brew           2.4G
                                  Subtotal               ~7.1G

Stale project artifacts           recsys-mvp/.venv        2.1G
  (last commit > 90d)             rt-survey/node_modules  727M
                                  ...
                                  Subtotal               ~8.3G

iOS simulators                                            3.3G
Docker volumes                    (74 unused volumes)    21.7G
─────────────────────────────────────────────────────────────
TOTAL                                                    ~40G

Deferred (needs action from you):
  Chrome cache: 1.5G — close Chrome first
  Docker images: XG — confirm separately (aggressive)
  Downloads/Movies/miniconda3/~/.local: XG — manual review
```

---

## Step 4: Confirm then execute

Ask for confirmation. Once confirmed, execute in this order (fast/safe first, slow/aggressive last):

1. Safe caches — `rm -rf` dirs + `brew cleanup` + `npm cache clean --force` + `pnpm store prune`
2. Stale project artifacts — `rm -rf` node_modules / .venv / .next per project
3. iOS simulators — `xcrun simctl delete unavailable`
4. Docker volumes — `docker volume prune -f`
5. Remind about deferred items

---

## Step 5: Report

```bash
df -H /System/Volumes/Data | tail -1
```

Report **before → after**, total freed. List any skipped items and why.
